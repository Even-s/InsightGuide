# 動態響應式佈局系統

**日期**: 2026-05-29  
**功能**: 根據螢幕尺寸和投影片方向自動調整佈局

---

## 系統概述

### 設計目標
自動偵測螢幕寬度和投影片方向，動態調整：
- ✅ 投影片區域寬度
- ✅ 卡片區域寬度和高度
- ✅ 建議逐字稿區域高度
- ✅ 轉錄區域高度
- ✅ 卡片尺寸和排列方向

### 核心原則
1. **響應式優先**: 根據可用空間最佳化佈局
2. **內容適應**: 不同螢幕展示不同資訊密度
3. **平滑過渡**: 所有變化都有動畫效果
4. **智能配置**: 考慮投影片方向的特殊需求

---

## 佈局模式

### 4 種螢幕尺寸模式

| 模式 | 螢幕寬度 | 典型裝置 | 說明 |
|------|----------|----------|------|
| **Compact** | < 1280px | 13" 筆電 | 緊湊佈局，優先顯示投影片 |
| **Standard** | 1280-1536px | 15" 筆電 | 標準佈局，平衡各區域 |
| **Wide** | 1536-1920px | 24" 螢幕 | 寬螢幕佈局，增大卡片區 |
| **Ultrawide** | ≥ 1920px | 27"+ 或雙螢幕 | 超寬佈局，最佳化全部區域 |

### 2 種投影片方向

| 方向 | 說明 | 卡片佈局 |
|------|------|----------|
| **Landscape** (橫向) | 一般簡報 | 垂直排列（由上到下）|
| **Portrait** (直立) | 直立簡報 | 水平排列（由左到右）|

**總計**: 4 × 2 = **8 種佈局配置**

## 技術實作

### useResponsiveLayout Hook

**位置**: `frontend/src/hooks/useResponsiveLayout.ts`

**功能**:
1. 偵測螢幕寬度變化
2. 根據投影片方向選擇配置
3. 返回完整的 LayoutConfig

**使用方式**:
```typescript
const { layoutConfig } = useResponsiveLayout(slideOrientation)

// layoutConfig 包含:
{
  slideArea: { width: 'flex-1' },
  cardsArea: { width: 'w-96', height: 'flex-1' },
  scriptArea: { height: 'h-80' },
  transcriptArea: { height: 'h-16' },
  cardsLayout: {
    direction: 'vertical',
    cardHeight: 'h-64',
    cardWidth: 'w-64'
  }
}
```

### 自動偵測機制

**螢幕寬度偵測**:
```typescript
useEffect(() => {
  function updateLayoutMode() {
    const width = window.innerWidth
    
    if (width < 1280) setLayoutMode('compact')
    else if (width < 1536) setLayoutMode('standard')
    else if (width < 1920) setLayoutMode('wide')
    else setLayoutMode('ultrawide')
  }
  
  updateLayoutMode()
  window.addEventListener('resize', updateLayoutMode)
  
  return () => window.removeEventListener('resize', updateLayoutMode)
}, [])
```

**投影片方向偵測**:
```typescript
// SlideViewer 中偵測
const handleImageLoad = (e) => {
  const img = e.currentTarget
  const aspectRatio = img.naturalWidth / img.naturalHeight
  const orientation = aspectRatio > 1 ? 'landscape' : 'portrait'
  onOrientationChange?.(orientation)
}
```

## 配置參數說明

### slideArea (投影片區域)

| 模式 | Landscape | Portrait |
|------|-----------|----------|
| Compact | `flex-1` | `flex-[0_0_45%]` |
| Standard | `flex-1` | `flex-[0_0_50%]` |
| Wide | `flex-[0_0_60%]` | `flex-[0_0_50%]` |
| Ultrawide | `flex-[0_0_55%]` | `flex-[0_0_45%]` |

**說明**:
- `flex-1`: 彈性擴展，填滿剩餘空間
- `flex-[0_0_50%]`: 固定 50% 寬度

### cardsArea (卡片區域)

**寬度**:

| 模式 | Landscape | Portrait |
|------|-----------|----------|
| Compact | `w-80` (320px) | `w-[55%]` |
| Standard | `w-96` (384px) | `w-[50%]` |
| Wide | `w-[40%]` | `w-[50%]` |
| Ultrawide | `w-[25%]` | `w-[55%]` |

**高度**:

| 模式 | Landscape | Portrait |
|------|-----------|----------|
| Compact | `flex-1` | `flex-[0_0_65%]` |
| Standard | `flex-1` | `flex-[0_0_60%]` |
| Wide | `flex-[0_0_60%]` | `flex-[0_0_55%]` |
| Ultrawide | `flex-1` | `flex-[0_0_50%]` |

### cardsLayout (卡片佈局)

**方向**:
- Landscape: `vertical`（垂直排列）
- Portrait: `horizontal`（水平排列）

**卡片高度**:

| 模式 | Landscape | Portrait |
|------|-----------|----------|
| Compact | `h-56` (224px) | `h-56` (224px) |
| Standard | `h-64` (256px) | `h-64` (256px) |
| Wide | `h-72` (288px) | `h-72` (288px) |
| Ultrawide | `h-80` (320px) | `h-80` (320px) |

**卡片寬度** (僅 Portrait):

| 模式 | 寬度 |
|------|------|
| Compact | `w-56` (224px) |
| Standard | `w-64` (256px) |
| Wide | `w-72` (288px) |
| Ultrawide | `w-80` (320px) |

## 動畫效果

### 平滑過渡

所有佈局變化都有過渡動畫：

```tsx
className="transition-all duration-300"
```

**效果**:
- 視窗大小變化時，區域寬度平滑過渡
- 投影片切換時，佈局平滑調整
- 卡片尺寸變化時，平滑縮放

**時長**: 300ms（剛好，不會太慢也不會太快）

## 響應流程

### 完整流程圖

```
使用者調整視窗大小
        ↓
window resize 事件觸發
        ↓
計算新的 layoutMode
(compact/standard/wide/ultrawide)
        ↓
根據 layoutMode + slideOrientation
查找對應配置
        ↓
更新 layoutConfig
        ↓
React 重新渲染
        ↓
CSS transition 動畫
        ↓
佈局平滑切換完成
```

### 投影片方向變化

```
切換投影片
        ↓
圖片載入
        ↓
計算長寬比
        ↓
判斷 orientation
(landscape/portrait)
        ↓
觸發 onOrientationChange
        ↓
更新 slideOrientation state
        ↓
useResponsiveLayout 重新計算
        ↓
佈局自動調整
```
